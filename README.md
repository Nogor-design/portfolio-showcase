# A1 Portfolio Program Manager

This folder is the command center for building and publishing Eric Irwin's software portfolio.

The portfolio should present existing work as polished, safe, recruiter-readable proof:

- AI systems and local-first agent workflows
- Evidence-grounded RAG and document automation
- Business operations dashboards
- Trading analytics, optimization, and simulation infrastructure
- Backend/data pipelines and desktop/runtime integration

## Working Principle

Do not invent portfolio projects until existing projects have been inspected, redacted, packaged, and turned into clear case studies.

## PM Files

| File | Purpose |
|---|---|
| `PROJECT_CHARTER.md` | One-week mission, scope, delivery standards, and risks |
| `PORTFOLIO_INVENTORY.md` | Candidate projects, demo surfaces, maturity, and packaging priority |
| `AGENT_TASK_LEDGER.md` | Work queue for Codex, Claude, Gemini, and Ollama/local models |
| `MODEL_ROUTING_AND_TOKEN_BUDGET.md` | Which model type gets which work, plus budget rules |
| `REDACTION_POLICY.md` | What must be removed or sanitized before anything becomes public |
| `WEEK_ONE_RUNBOOK.md` | Day-by-day execution plan |
| `WEEK_TWO_RUNBOOK.md` | Public deployment, sharing, polish, and outreach plan |
| `WEEK_THREE_RUNBOOK.md` | Recruiter-facing polish, worker coordination, and sixth-case gate |
| `WEEK_FOUR_RUNBOOK.md` | Sixth-case public-safe demo package and verification |

Current expansion work also includes a Week 5 candidate pass adding the TraderDan
client site plus MA-Cross and Large Candle strategy-analysis workbench cases.

## Delivery Target

A usable portfolio site with 7-10 strong case studies, each containing:

- Problem solved
- What was built
- Architecture overview
- Demo screenshots or video
- Technical highlights
- Business value
- Safety/redaction note where relevant
