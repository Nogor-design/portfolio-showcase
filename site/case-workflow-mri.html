Problem

Many useful internal-tool opportunities start as a hard-to-explain pile of files, screenshots, notes, approvals, handoffs, and exceptions. The buyer needs clarity before they can even decide what software should exist.

Solution

Workflow MRI makes the first useful artifact visible: a reconstructed process graph with owners, cycle times, bottlenecks, risk findings, review tasks, ranked automation candidates, and an exportable handoff package.

Where This Fits

Lead with this for AI engineering, operations automation, workflow discovery, and internal-tool conversations. It shows the full pattern: messy inputs, evidence, governance, human review, simulation, and a product surface people can inspect.

What It Proves

  • React product design around a real operating workflow, not a marketing mockup.
  • Typed artifact contract between a Python engine and a static public renderer.
  • Public-safe AI posture: deterministic replay, fictional data, and no browser-side inference.
  • Governance features that buyers expect: Safe Mode, evidence links, risk findings, and review tasks.

Verification

Current audit: `npm run build` passes for the static web app, `npm test` passes 2 Safe Mode redaction tests, and `python -m pytest` passes 2 schema smoke tests. Browser verification loaded the workbench, opened the Process Graph, confirmed Safe Mode ON, and found no console warnings or errors.

Build Status

Phase 1 is portfolio-ready: the hand-authored Meridian Claims bundle renders end to end. The next implementation milestone is Phase 2, where `make bundle` regenerates that bundle from fictional sample inputs through the local Python engine.

Redaction Note

Public materials use the fictional Meridian Claims company only. No customer files, private local paths, secrets, accounts, or real personal data are part of the demo.
