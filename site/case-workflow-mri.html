Architecture + proof

Messy artifacts become a governed workbench

The public demo is a deterministic static renderer over a fictional Meridian Claims bundle generated by the local Python engine. The engine ingests six messy source artifacts, mines a directly-follows process graph, detects governance risks, ranks automations, simulates before/after impact, emits an audit trail, and runs a critic pass.

Workflow Architecture

1

Chaotic Inputs

Spreadsheets, PDFs, emails, screenshots, notes, and exports.

2

Local Engine

Process-mining and rule engines generate steps, edges, risks, reviews, automations, simulation, and audit events.

3

Critic + Evals

A critic validates evidence and counts; evals score steps, edges, risks, PII, bottleneck, and hallucination rate.

4

Static Workbench

React renders the generated Artifact Bundle with no browser-side inference, keys, or private data.

Problem

Many useful internal-tool opportunities start as a hard-to-explain pile of files, screenshots, notes, approvals, handoffs, and exceptions. The buyer needs clarity before they can even decide what software should exist.

Solution

Workflow MRI makes the first useful artifact visible: a reconstructed process graph with owners, cycle times, bottlenecks, risk findings, review tasks, ranked automation candidates, audit events, before/after simulation, and an exportable handoff package.

Where This Fits

Lead with this for AI engineering, operations automation, workflow discovery, and internal-tool conversations. It shows the full pattern: messy inputs, evidence, governance, human review, simulation, and a product surface people can inspect.

What It Proves

  • Deterministic process mining from a fictional claims event log, not a prompt-only demo.
  • Typed artifact contract between a Python engine and a static public renderer.
  • Governance features that buyers expect: Safe Mode, evidence links, risk findings, review tasks, audit events, and critic checks.
  • Public-safe AI posture: fictional data, optional LLM enrichment, and no browser-side inference.

Verification

Current audit: the Workflow MRI repo reports 19 engine tests across schema, pipeline, agents, critic, audit, and eval coverage, plus 2 web redaction tests. Evals score 100% step recall, 100% edge-kind recall, 100% risk-category recall, 100% PII recall, and a 0.0% hallucination rate against independently-authored ground truth.

Build Status

Phase 2 and the core Phase 3 governance loop are now implemented for the Meridian Claims demo: `make bundle` regenerates the bundle locally, `make promote` publishes it into the static web renderer, and the Export view includes the generated audit trail. Remaining polish is a short demo video and a second sample company.
