Problem

Trading automation needs a careful runtime boundary: live platform state, local operator visibility, strategy commands, safety checks, and audit events all need to meet without exposing private accounts or letting automation bypass human control.

Solution

The app runs a Python WebSocket server for NinjaTrader-originated data, keeps a thread-safe AppState store for the DearPyGUI interface, and publishes strategy events through a localhost JSON-lines API. The GUI separates dashboard monitoring from the strategy operations console.

What It Proves

  • Desktop/runtime integration across NinjaScript, Python, WebSockets, and a GUI render loop.
  • Explicit boundaries between data transport, state storage, event bus, strategy API, and UI panels.
  • Operator-centered safety gates for intake, heartbeat, recovery, flatten, cancel, and live confirmation.
  • Testing discipline around direct strategy API commands and fake NinjaTrader clients.
