Empire of Drawdown

A trading-strategy game that turns deterministic replay, risk, and strategy outcomes into a browser walkthrough a reviewer can understand quickly.

Problem

Strategy replay systems are hard to inspect when they are presented only as logs, CSVs, or backend test results. This project wraps the same kind of deterministic infrastructure in a game layer so a reviewer can see the flow from configuration to replay to debrief.

Solution

The app uses a layered TypeScript monorepo: simulator, strategy engine, game domain, Fastify API, and React/Vite UI. Seeded demo scenarios let the product be reviewed without live brokerage connections or private trading accounts.

What It Proves

  • Clear architecture boundaries across simulation, domain, API, and UI.
  • Deterministic replay and testable strategy execution.
  • Product thinking: seeded reviewer scenarios, battle replay, and debrief screens.
  • Engineering restraint around what is real versus intentionally stubbed.

Verification

Current audit: Node and npm available, dependencies installed, `npm.cmd test` passed with 169 tests and 8 skipped. Demo-mode API and Vite web launch both returned HTTP 200.
