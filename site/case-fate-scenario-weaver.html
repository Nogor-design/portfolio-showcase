Fate Scenario Weaver

A local AI-assisted workbench for building Fate RPG scenarios from campaign data, player characters, aspects, top skills, clocks, clues, NPC pressure, audits, and exportable play materials.

Problem

Tabletop scenario prep can drift into generic hooks that do not use the actual player characters. The tool needs to make each PC matter, weave top skills and aspects into the scenario, and keep generated output structured enough for refinement and export.

Solution

Fate Scenario Weaver stores campaigns, characters, scenario versions, audits, and structured JSON in SQLite through Prisma. The app can run with deterministic mock AI, Ollama, or CLI-backed providers, while preserving a shared staged architecture for seeds, full scenarios, audits, refinement, GM guides, scene detail, and exports.

What It Proves

  • Provider boundaries for local mock, Ollama, and CLI-backed AI generation.
  • Structured generation with repair logic when outputs drift away from the selected party.
  • Domain modeling across campaigns, Fate characters, skills, NPCs, locations, scenes, clues, compels, clocks, audits, and versions.
  • Export surfaces for Markdown, JSON, and PDF play materials.

Current Evidence

The portfolio capture used a temporary public-safe SQLite database seeded from the project sample data with `AI_PROVIDER=mock`. `npm run lint` and `npm run build` both passed after a small JSX/import cleanup.

Redaction Note

Public materials use seeded fictional campaign data only. The app's `.env` files, user-created campaign database, generated export files, CLI provider settings, and private image-generation outputs are not copied into the portfolio.
