## Portfolio Story

Trades Resource Command Center is a staffing and recruiting operations dashboard that replaces spreadsheet, paper, and memory-based tracking with a daily command center. It helps an owner or operations lead see urgent assignments, ending placements, open orders, recruiter follow-up, candidate readiness, and customer-review status from one place.

This is the strongest contract-work case study because it shows practical software for a real business workflow.

## What It Proves

- Business workflow analysis
- Dashboard design for daily operations
- Streamlit application development
- Customer-upload staging and review
- Privacy-aware demo/session handling
- Data normalization from messy operational files
- Owner/admin versus customer-review mode design

## Risks And Redactions

- Never publish real customer spreadsheets, PDFs, candidates, clients, names, phone numbers, emails, or uploaded source files.
- Use generated demo data for every public image.
- Keep raw upload folders, runtime sessions, and local secrets out of Git and screenshots.
- Frame the app as a prototype/customer-review workflow unless production persistence has been built.
- Avoid showing hidden owner/admin controls in customer-facing screenshots unless the case study is explaining security/session architecture.

## Case Study Outline

1. Problem: staffing operations are split across spreadsheets, paper tracking, and memory.
2. Solution: a focused daily command center for the owner and recruiters.
3. Workflow: owner view, forecast, open orders, recruiter process, import review.
4. Data handling: generated demo data plus temporary customer upload sessions.
5. Business value: fewer missed renewals, faster follow-up, clearer assignment and order visibility.
6. Production path: database persistence, authentication, outbound communication, client review links.
